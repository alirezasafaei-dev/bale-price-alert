## مراحل بعدی

### فاز ۱: اصلاح قیمت‌ها (اولویت بالا)

**هدف**: نمایش قیمت واقعی ارزهای کوچک و قابل خرید از بازار ایران

**راه‌حل پیشنهادی**:
1. ساخت یک proxy ساده در VPS ایران
2. Proxy فقط یک endpoint دارد: `/api/prices`
3. Worker از این proxy قیمت می‌گیرد
4. نمایش ارزهای کوچک: USDT, DOGE, SHIB, TRX, ADA, DOT

**پیاده‌سازی**:
```bash
# در VPS ایران
# ساخت یک سرویس Node.js ساده که:
# - از Nobitex یا Wallex قیمت می‌گیرد
# - فقط /api/prices را expose می‌کند
# - با PM2 اجرا می‌شود
```

**ارزهای پیشنهادی**:
- USDT (تتر) - پایه‌ای‌ترین
- DOGE (دوج‌کوین) - محبوب و ارزان
- SHIB (شیبا) - محبوب و ارزان
- TRX (ترون) - محبوب
- ADA (کاردانو) - محبوب
- DOT (پولکادات) - محبوب

### فاز ۲: اصلاح فلوی هشدار

**مشکل**: کاربر می‌تواند بدون طی مراحل، مستقیم عدد بفرستد

**راه‌حل**:
1. اصلاح `handleTextInSession` برای چک کردن session کامل
2. اگر session ناقص است، راهنمایی کاربر
3. اضافه کردن دکمه "لغو" در هر مرحله

### فاز ۳: تست و بهینه‌سازی

1. تست کامل فلوی هشدار
2. تست cron job
3. تست نمایش قیمت‌ها
4. بهینه‌سازی پیام‌ها
